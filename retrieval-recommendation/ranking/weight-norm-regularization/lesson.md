# 课程：范数为何能约束推荐模型，而不只是把数字变小

## 1. 从“长度”到模型复杂度

向量 `v = [v1, ..., vd]` 是一组有方向的数。它可以是一次曝光的特征、一个 user/item 的 embedding，或一层模型的参数。**范数**是把向量映射为非负“大小”的规则，记作 `||v||`；它必须满足：

1. `||v|| = 0` 当且仅当 `v = 0`；
2. `||a v|| = |a| ||v||`（整体放大 a 倍，长度也放大）；
3. `||u + v|| <= ||u|| + ||v||`（三角不等式）。

常见的 Lp 范数是 `||v||p = (sum_i |vi|^p)^(1/p)`。因此：`L1 = sum_i |vi|`，`L2 = sqrt(sum_i vi^2)`，`L∞ = max_i |vi|`。例如 `v=[3,4]` 的 L1 是 7、L2 是 5、L∞ 是 4。p 不是“向量有几个维度”；同一个 128 维 embedding 可以计算多种范数。

范数回答的是“整体有多大”，不是“语义是否相似”。在推荐中，余弦相似度还会除掉两端 L2 范数，主要比较方向；点积 `u^T v` 同时受方向和两端长度影响。

## 2. 矩阵的两种常见大小

矩阵 `W` 是把输入特征变成下一层表示的线性规则：`h = W x + b`。若 `x` 有 d 个特征、`h` 有 k 个输出，`W` 的 shape 为 `[k, d]`；`Wij` 表示第 j 个输入特征对第 i 个输出的权重。

最常用于神经网络权重衰减的是 **Frobenius 范数**：

`||W||F = sqrt(sum_i sum_j Wij^2)`。

它等同于把所有元素摊平成一个长向量后取 L2，易算、易求导，所以经常写成 `||W||F^2`。但它不直接表示任意输入会被放大多少。

若问题是“这层最多能把输入放大多少”，用 **诱导（算子）范数**：

`||W||p = max_(x != 0) ||W x||p / ||x||p`。

其中 L2 诱导范数是谱范数 `||W||2`，等于 W 最大奇异值。它给出边界 `||W x||2 <= ||W||2 ||x||2`，因此更直接约束局部扰动在这层被放大的上界。还有核范数（奇异值之和），常用来偏好低秩矩阵。不要把三者混为一谈：Frobenius 惩罚所有参数总能量，谱范数限制最强放大方向，核范数与低秩偏好相关。

## 3. 正则化不是另一个评估指标，而是训练目标的一部分

没有正则化时，训练只最小化数据损失：

`min_theta L_data(theta)`。

加入 L2 权重正则化后变成：

`min_theta L_data(theta) + (lambda / 2) ||theta||2^2`。

`theta` 可以是一层的 `W`，也可以是全部要训练的 embedding 和 MLP 权重。`lambda >= 0` 是权衡系数：大 λ 更偏好小范数，小 λ 更优先贴合训练数据。它不保证测试集必然更好；λ 需要在验证集上选，并受样本量、噪声、模型、训练轮数影响。

为什么这会是正则化？考虑线性打分 `s = w^T x`。对 L2 有 Cauchy–Schwarz 不等式：`|s| <= ||w||2 ||x||2`。当输入特征尺度受控时，小 `||w||2` 直接限制模型能输出多极端的分数，也限制分数对特征扰动的敏感性：`|w^T (x-x')| <= ||w||2 ||x-x'||2`。为记住一条偶然的或噪声很大的样本而把某些权重推得很大，会付出明确代价。优化器因此在“降低样本误差”和“保持较平滑/较不极端的函数”之间权衡。

等价的约束写法是 `min L_data(theta), subject to ||theta||2^2 <= C`：在适当的正则条件下，它与某个 λ 的惩罚写法有相同解。特别地，如果有许多训练误差都同样小的解，L2 会偏好其中最小范数者。这里的“最小范数”是**在拟合数据的候选解中选一个**，绝不是“不管数据，直接把权重设为零”。

L1 则使用 `lambda ||theta||1`。它的等值面有轴对齐的尖角，最优解更容易落在某些坐标为零的位置，因而常带来稀疏参数；L2 的平方惩罚通常连续地缩小每个参数，但不常使它精确为零。

## 4. 它怎样进入一次训练更新

对 L2 目标求导：

`grad_total = grad_data + lambda * theta`。

普通 SGD 的一步便是：

`theta <- theta - eta * grad_total = (1 - eta*lambda) theta - eta * grad_data`。

所以每一步都有一个把参数朝零收缩的因子 `(1 - eta*lambda)`，这就是常说的 weight decay。若某参数没有来自数据的梯度，它仍会被收缩（取决于优化器对该参数组的具体实现）。

对普通 SGD，L2 penalty 与这种形式对应；对 Adam 一类有自适应预条件的优化器，把 L2 梯度混进 adaptive update 不再严格等同于“每步按固定比例衰减”。AdamW 把衰减从 adaptive gradient update 中解耦，直接做参数收缩。工程上也常把 bias、LayerNorm 的 scale/bias 排除出 weight decay；embedding 是否衰减应作为超参组选择，而不是机械地“一律衰减”。

## 5. 推荐系统：特征、参数、embedding 各在哪里

一次排序样本可写为：

`x = [年龄段, 设备, 最近点击数, query 类目, ...]`，它是 **feature vector**。经过编码和特征交叉后，模型用参数矩阵 `W1, W2, ...` 计算 click logit；这些矩阵是学到的规则。

高基数离散 ID（user_id、item_id、作者_id、类目_id）通常不做 one-hot 后直接乘巨矩阵，而是查 embedding table `E`。`E` 的 shape 是 `[num_items, d]`，第 i 行 `E[i] = v_i` 是 item i 的 d 维参数向量。它既像“item 的语义表示”，在训练时又是需要优化和可能需要正则化的权重。

双塔召回的极简打分是 `s(u,i) = e_user[u]^T e_item[i]`。若某 item 仅有很少交互，模型可能靠把 `e_item[i]` 的 L2 范数推大来提高少数正样本的点积，造成不稳定的热门度/置信度信号。可用的目标之一是：

`L = L_pair_or_click + (lambda/2) (||U||F^2 + ||V||F^2)`。

U、V 分别是所有 user 和 item embedding 组成的矩阵；Frobenius 项正是“每一行 embedding 的 L2 平方之和”。它会抑制只靠放大向量长度记住少量样本的捷径，迫使模型更多利用稳定的方向结构和共享模式。注意这不是绝对禁止高分：让用户和物品方向一致仍能得到高点积；只是高范数也要付出代价。

一个额外的重要性质是缩放歧义：`(c*u)^T (v/c) = u^T v`，所以不正则化时同一分数可由很多不同长度的因子给出。对 `||u||2^2 + ||v||2^2` 的惩罚会偏好更平衡的尺度，避免一个塔无限变大、另一塔无限变小。若线上使用 cosine similarity，向量会归一化，长度不直接参与检索分数；但训练中的范数仍影响梯度、数值稳定和模型可学到的表示，是否施加范数约束仍要按训练目标验证。

## 6. 常见误解、边界和验证

- **误解：范数就是平均值。** 它衡量大小；均值可能为零而 L2 很大。
- **误解：正则化直接修改输入特征。** L2 weight decay 通常修改的是参数；feature normalization 是另一件事，用于让输入尺度可比较。
- **误解：范数越小，泛化一定越好。** λ 太大会欠拟合；深网中范数只是复杂度的一个有用代理，不是唯一原因。
- **误解：所有矩阵范数都可以互换。** 它们刻画不同约束；选择需与目标（总参数规模、Lipschitz 放大、低秩）相配。
- **边界：** 对稀疏 embedding，正则化是否作用于未访问行、如何与负采样及 embedding 归一化配合，取决于框架和实现；应检查实际 optimizer/update。
