# 盲点自测 — python


- 写一段代码复现浅拷贝共享内层列表的问题，并分别用数据重构和深拷贝修复。
- 实现带参数的 @retry(max_attempts=3) 装饰器，只捕获调用方指定的异常类型。
- 分别用线程池和进程池执行 I/O 等待任务、纯 Python CPU 任务，记录并解释耗时差异。
- 用 cProfile 分析一个脚本，找出累计耗时最高的三个函数；优化其中一个并重新测量。
- 写一个持有文件句柄的 C++ RAII 类，禁用复制并支持移动。
- 故意制造一次堆越界，分别观察普通运行和 AddressSanitizer 的报错差异。
- 把一个 C++ 库拆成头文件、源文件和可执行程序，用 target-based CMake 构建。
- 用 ldd/otool -L 与 nm -C 分析一个 Python 原生扩展的依赖和导出符号。
- 在干净虚拟环境中安装 PyTorch，记录 sys.executable、框架版本、构建 CUDA 版本和设备可见性。
- 改造综合实战：接收 NumPy 数组，检查 dtype 与连续性，并比较复制版和共享缓冲区版的耗时。
